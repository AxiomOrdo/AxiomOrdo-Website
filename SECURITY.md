# Security Policy

## Supported Versions

AxiomOrdo is a web platform delivered as a continuously deployed service. Security updates are applied to the current production release only. There are no legacy versions in active use.

| Version | Supported |
| ------- | --------- |
| Current production (axiomordo.com) | ✅ |
| Previous production builds | ❌ |
| Development / preview branches | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability in the AxiomOrdo platform, website, or associated infrastructure, please report it responsibly.

**Contact:** [security@axiomordo.com](mailto:security@axiomordo.com)

Please include in your report:
- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept (where safe to provide)
- The URL, endpoint, or component affected
- Any relevant screenshots or supporting evidence

### What to expect

| Stage | Timeframe |
| ----- | --------- |
| Initial acknowledgement | Within 48 hours |
| Triage and severity assessment | Within 5 business days |
| Resolution or workaround | Depends on severity — critical issues are prioritised immediately |
| Notification of fix | Once remediation is confirmed |

### Disclosure policy

We ask that you:
- Give us reasonable time to investigate and remediate before public disclosure
- Do not access, modify, or exfiltrate data beyond what is needed to demonstrate the vulnerability
- Do not perform denial-of-service testing or social engineering against AxiomOrdo personnel

We will not take legal action against researchers who report vulnerabilities in good faith and follow this policy.

### Scope

In scope:
- axiomordo.com and subdomains
- Any AxiomOrdo-operated API endpoints or infrastructure
- Authentication and session handling
- Data handling and evidence chain integrity

Out of scope:
- Third-party services or platforms integrated with AxiomOrdo (report to the relevant vendor)
- Issues in end-of-life dependencies with no viable upgrade path and no exploitable attack vector in our context
- Findings from automated scanners without demonstrated impact
